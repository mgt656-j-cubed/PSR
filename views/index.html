Sprint Completed Stories

The following stories have been completed as of Sprint 1:

  • Site looks pretty with Bootstrap CSS
  • Site looks awesome with header and footerelements
  • Site is easily navigable with a link to the about page and the homepage in the footer element of the page
  • Added a team logo to the homepage, which is found at /
  • Homepage displays an unordered list of events, including their details (with)
  • Events that are over are hidden
  • Events are linked to event detail page (currently unpopulated)
  • Created an API link to retrieve an object with key events containing all events in JSON format
  • Events can be searched by submitting a GET request
  • Events can be created at /events/new(form is shown, but cannot be submitted or validated)

We started relatively early with the project. We had our first sprint planning on Thursday, January 29. During this meeting, we went through each and every user story, elaborated on the work that needed to be done, agreed on a definition of done, assigned "difficulty levels"/story points for each, and then split the work among ourselves. At the same meeting, we set down our schedules for (virtual) daily standup meetings, a retrospective, and a sprint demo.

From the list above, you'll probably notice that we have finished quite a good amount of work during this sprint, and this is due to our (mis)understanding of the project instructions. When we saw S1-A to S1-F in the project site, we assumed that all those user stories were due for Sprint 1. It was only much later when we asked the TAs and realized that videos to complete some of the user stories have not yet been posted that we were in fact moving way faster than was needed.

We struggled a lot with merging branches, and that hampered us from moving forward with the project. All of us are fairly new with Git (and even programming in general), so the learning curve was to be expected.
